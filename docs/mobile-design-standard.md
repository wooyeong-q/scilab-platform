# SciLab 모바일·디자인 품질 기준

## 목표

SciLab과 연결된 모든 과학 프로그램은 휴대폰, 태블릿, PC에서 수업의 핵심 활동을 완료할 수 있어야 한다. 특정 기기나 마우스에만 의존하는 기능은 공개하지 않는다.

## 공통 화면 규격

| 구간 | 기준 폭 | 사용 목적 |
|---|---:|---|
| 소형 모바일 | 360px 이상 | 작은 Android, iPhone mini급 |
| 일반 모바일 | 390~430px | 현재 iPhone·Android 기본 검수 |
| 태블릿 | 768~1024px | 세로·가로 수업 환경 |
| PC | 1280px 이상 | 교사용 화면과 대형 모니터 |

- 콘텐츠의 최소 지원 폭은 360px이다.
- 320~359px에서는 핵심 기능을 유지하되 일부 설명을 축약할 수 있다.
- 가로 스크롤은 지도·넓은 데이터 표처럼 의도된 영역에서만 허용한다.
- 브라우저의 주소창 변화에는 `dvh`, 홈 인디케이터와 노치에는 `safe-area-inset-*`를 사용한다.
- 사용자의 화면 확대를 막는 `user-scalable=no`와 `maximum-scale=1`을 사용하지 않는다.

## 조작과 접근성

- 버튼, 링크, 입력 컨트롤의 터치 영역은 최소 44×44px이다.
- 모바일 입력 글자는 16px 이상으로 설정하여 iOS 자동 확대를 방지한다.
- 드래그 활동에는 `항목 선택 → 대상 터치` 방식의 대안을 제공한다.
- 색상만으로 상태를 구분하지 않고 글자, 아이콘 또는 형태를 함께 사용한다.
- 모든 입력에는 보이는 라벨을 제공하고 오류는 화면과 보조기술에 함께 알린다.
- 키보드 포커스 표시를 제거하지 않는다.
- `prefers-reduced-motion` 환경에서는 불필요한 움직임을 줄인다.

## 화면 방향

- 사이트와 프로그램의 핵심 기능은 세로 화면에서 사용할 수 있어야 한다.
- 넓은 실험 무대가 필요한 프로그램은 가로 화면을 권장할 수 있지만 회전을 강제하지 않는다.
- 방향 전환 후 선택 상태, 입력값과 실험 진행 상태가 유지되어야 한다.
- iOS에서 화면 방향 잠금 API가 실패할 수 있으므로 사용자가 직접 회전할 수 있는 안내를 제공한다.

## 디자인 토큰

| 역할 | 값 | 용도 |
|---|---|---|
| 배경 | `#f5f7fb` | 전체 페이지 |
| 기본 표면 | `#ffffff` | 카드·폼·팝업 |
| 기본 글자 | `#122033` | 제목·본문 |
| 보조 글자 | `#607087` | 설명·통계 |
| 주 색상 | `#5156e8` | 브랜드·선택 상태 |
| 보조 색상 | `#0891b2` | 과학·탐구 포인트 |
| 기본 테두리 | `#dfe5ef` | 카드·입력창 |

- 카드 모서리는 18~26px, 입력창과 버튼은 12~14px을 기본으로 한다.
- 그림자는 정보 계층을 표현할 때만 사용하고, 테두리와 함께 과도하지 않게 적용한다.
- 제목은 굵고 간결하게, 설명은 충분한 행간으로 읽기 쉽게 구성한다.

## 외부 프로그램 연결 계약

플랫폼의 실행 화면은 iframe 바깥 영역만 제어한다. 연결 프로그램은 자체적으로 다음 조건을 충족해야 한다.

- 반응형 viewport 설정
- 360px 세로 화면의 핵심 활동 완료
- 터치와 마우스 동시 지원
- 모바일 키보드가 입력창이나 제출 버튼을 가리지 않음
- iframe 실행과 새 창 실행 모두 지원
- 로딩, 네트워크 실패와 저장 실패 안내 제공
- 배포 주소가 고정된 운영 별칭을 사용

## 단계별 검수표

| 화면 | 360×800 | 390×844 | 768×1024 | 1366×768 |
|---|---|---|---|---|
| 메인·검색·필터 | 필수 | 필수 | 필수 | 필수 |
| 프로그램 상세 | 필수 | 필수 | 필수 | 필수 |
| 프로그램 실행 | 세로·가로 | 세로·가로 | 세로·가로 | 필수 |
| 제안 폼 | 키보드 포함 | 키보드 포함 | 필수 | 필수 |
| 관리자 화면 | 필수 | 필수 | 필수 | 필수 |

운영 반영 전에는 최소한 iOS Safari, Android Chrome, Samsung Internet을 확인한다. 학교에서 링크를 메신저로 배포한다면 카카오톡 인앱 브라우저도 확인한다.

## 완료 조건

다음 조건을 모두 충족해야 모바일 대응 완료로 표시한다.

1. 세로·가로 화면에서 핵심 활동을 끝낼 수 있다.
2. 화면 잘림, 비의도성 가로 스크롤과 터치 충돌이 없다.
3. 마우스 없이 모든 핵심 조작이 가능하다.
4. 태블릿·PC의 기존 기능이 퇴행하지 않는다.
5. 타입 검사와 프로덕션 빌드가 통과한다.
6. 미리보기 배포에서 실제 기기 검수를 완료한다.
